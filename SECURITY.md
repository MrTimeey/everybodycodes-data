## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public issue**.

Instead, contact:

**coding@mrtimeey.com**

Include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- A minimal proof-of-concept if applicable

You will receive a confirmation within 48 hours and, if validated, updates as the issue is resolved.

---

## Disclosure Process

1. Vulnerability is reported privately.
2. It is verified and a patch is prepared.
3. A new version is released to npm.
4. A public advisory is published once users have had time to upgrade.

We deeply appreciate responsible disclosure and will credit researchers accordingly.
